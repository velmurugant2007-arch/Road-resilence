# Deliverable 5: ATLAS Reusable Component Library

This document specifies the anatomical structure, states, and props for all reusable reusable UI components composing the ATLAS geospatial dashboard.

---

## 1. Global Navigation & HUD Header

### `TopTelemetryBar`
* **Anatomy:** Left-aligned ISRO/ATLAS branding emblem; center-aligned system health pill (`HEALTHY: <50ms`); right-aligned presentation action buttons: **Export Report**, **Capture Screenshot**, **Generate PDF**, **Settings**, and **Notification Bell** (with unread badge counter).
* **Styling:** Floating glassmorphic bar (`--surface-1`) anchored 16px from top viewport edge.

### `CommandPaletteModal` (`Ctrl+K` / `Cmd+K`)
* **Anatomy:** Linear.app style floating search dialog with instant fuzzy lookup.
* **Capabilities:** Search nodes (`N154`), edges (`E-088`), and repair IDs (`RH-001`); instantly jump across the 8 dashboard views; trigger stress simulations; export reports; execute common system actions.

### `MiniMapOverview`
* **Anatomy:** Compact bottom-left spatial inset card (`220px x 160px`) rendering Hero City extent, current viewport bounds rectangle, active disaster blast zone indicator, and live camera coordinates.

### `NotificationCenterDrawer`
* **Anatomy:** Slide-out glassmorphic panel storing chronological telemetry events (e.g., "Simulation completed", "Graph healing accepted RH-001", "AI inference completed", "GeoJSON Exported", "System Warnings").

### `SidebarNavigationDock`
* **Anatomy:** Collapsible left dock containing vertical icon buttons representing the 8 system views.
* **States:** Collapsed (64px width, icons only with tooltip hover) vs. Expanded (220px width with text labels). Controlled via Framer Motion spring transition.

---

## 2. Floating Analytics & Inspector Panels

### `FloatingInspectorPanel`
* **Anatomy:** Draggable/resizable frosted glass container overlaying the right 30% of the map canvas. Features an header with close/minimize controls and a scrollable body container.
* **Props:** `title: string`, `icon: ReactNode`, `isOpen: boolean`, `onClose: () => void`.

### `StatisticsCounterCard`
* **Anatomy:** Compact glass card displaying a top caption label, a prominent monospace animated number, and a bottom trend indicator ($\Delta$ percentage vs baseline).
* **Visual Polish:** Uses Framer Motion `useSpring` counter to smoothly tick upwards when incoming WebSocket or API data updates.

---

## 3. Specialized Domain Inspectors

### `ExplainabilityXAIPanel` & `RepairExplanationViewer`
* **Purpose:** Surfaces the backend `RepairExplanationModel` object whenever a user clicks a healed edge on the map.
* **Anatomy:**
  * **Header:** Edge ID (`RH-001`) with accepted status pill (`ACCEPTED` in green, `REJECTED` in red).
  * **Score Breakdown Gauge:** 5 visual horizontal progress bars showing normalized scores for AI Confidence, Direction Consistency, Distance, Width, and Density.
  * **Hybrid Cost Total:** Large glowing circle displaying final hybrid score (`0.93 / 0.65 threshold`).
  * **Human Rationale Box:** Frosted inner callout rendering the human-readable text explanation generated by the backend.

### `CriticalRoadInspector`
* **Purpose:** Surfaces multi-metric vulnerability analysis when a road link or junction is selected.
* **Anatomy:** Displays Betweenness Centrality percentile rank, K-Core shell index, and a warning badge if the node is identified as an **Articulation Bridge**.

### `DecisionSupportPanel`
* **Purpose:** Displays emergency road repair priority rankings after a simulated disaster.
* **Anatomy:** Ordered table/list showing Priority Rank (#1, #2...), Road Segment ID, and estimated marginal efficiency improvement ($\Delta R = +4.2\%$). Includes an interactive "HIGHLIGHT ROUTE" button to pan/zoom the WebGL map directly to the broken link.

---

## 4. Simulation & Control Widgets

### `SimulationTimelineHUD`
* **Anatomy:** Bottom-center floating floating playback HUD mimicking video editing software. Contains Play/Pause toggles, step-forward/backward buttons, and a scrubbing slider representing failure propagation stages.

### `ParameterSliderWidget`
* **Anatomy:** Slider track with glowing thumb thumb, numerical input box, and real-time debounced trigger. Used for setting AI Confidence Threshold (`0.0 - 1.0`), Search Radius (`10m - 500m`), or Disaster Failure Fraction (`0% - 50%`).
