# Project ATLAS — Route Resilience

## Occlusion-Robust Road Extraction & Graph-Theoretic Criticality Analysis for Urban Mobility

**Competition**: Bharatiya Antariksh Hackathon 2026 (ISRO)  
**Problem Statement**: PS-4 — Route Resilience  
**Version**: 0.1.0-alpha  
**Status**: Phase 1 — Workspace Initialization

---

## Mission

Build an ISRO-grade engineering system that extracts road networks from satellite imagery under occlusion conditions (cloud cover, tree canopy, building shadows), constructs graph-theoretic representations of the extracted networks, and performs criticality analysis to identify infrastructure vulnerabilities affecting urban mobility.

This project treats the hackathon problem as a professional engineering challenge — prioritizing scientific correctness, engineering rigor, and documentation quality over speed.

---

## Problem Domain

| Dimension | Description |
|---|---|
| **Computer Vision** | Satellite image segmentation under occlusion — roads obscured by clouds, vegetation, shadows |
| **Graph Theory** | Road network topology construction, criticality metrics, resilience analysis |
| **GIS** | Geospatial coordinate systems, projection handling, spatial indexing |
| **Urban Mobility** | Travel time estimation, alternative route analysis, infrastructure vulnerability |

---

## Engineering Principles

1. **Research before implementation** — Every algorithm selection is evidence-based
2. **Documentation equals code** — Every module traces to a requirement
3. **Scientific correctness** — No hallucinated metrics, no unjustified claims
4. **Traceability** — Every feature maps to an official requirement or documented innovation
5. **Quality over speed** — ISRO-grade engineering standards throughout
