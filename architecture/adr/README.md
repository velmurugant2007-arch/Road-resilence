# Architecture Decision Records

This directory contains all Architecture Decision Records (ADRs) for the project.

## What is an ADR?

An ADR documents a significant architectural decision — the context, options considered, decision made, and rationale. ADRs are never deleted; superseded ADRs are marked as such.

## Creating an ADR

1. Copy the [ADR template](../../templates/adr_template.md)
2. Name it `ADR-XXXX-brief-title.md` (e.g., `ADR-0001-graph-library-selection.md`)
3. Fill in all sections
4. Update `memory/decisions.md` with a summary
5. Update `memory/architecture_history.md`
