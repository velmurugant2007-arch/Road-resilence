# Testing Guide

## Project ATLAS — Route Resilience

> Testing strategy, test categories, coverage requirements, and quality assurance. Populated during Phases 5-9.

---

## Status: Awaiting Phase 5 (Architecture)

## Testing Philosophy

- Every module must be testable in isolation
- AI models are evaluated with quantitative metrics, not subjective assessment
- Graph algorithms are validated against known network topologies
- Integration tests verify module interactions
- No feature is "done" without tests

## Test Categories (to be populated)

1. Unit Tests — Individual functions and classes
2. Integration Tests — Module interactions and data pipeline
3. AI Evaluation — Model accuracy, occlusion robustness, inference speed
4. Graph Validation — Topology correctness, centrality accuracy
5. End-to-End Tests — Full pipeline from image to dashboard
6. Performance Tests — Inference speed, memory usage
